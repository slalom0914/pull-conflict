## 충돌사례

### CASE#1
원격레포와 로컬 레포의 커밋 이력이 같은 상황에서 로컬에서
working directory에서의 수정 이후 git pull한다면?

```bash
# pull = fetch + merge
# fetch는 성공인데 merge가 실패한 경우
git pull origin main
```
충돌이 아니므로 그냥 git push하면 된다.

### CASE#2
원격레포에서 commit 이력을 추가했을 때 충돌이 없다.
git pull origin main을 통해 commit이력을 내려 받은 후
병합의 일종으로 볼 수 있으나, 변경사항에 대한 head만
이동하므로 fast forward에 해당함.

### CASE#3
원격에 커밋이력 추가 발생시킴.
또 로컬에서도 별도의 커밋이력을 발생시킴.
단 같은 파일에 대한 수정이 아닐 때 -> 충돌 발생 안함.
git pull시 문제 없이 실행되고 단 fast forward가 아닌
머지 커밋이 일어난다.
이 경우 다시 push가 필요하다.
pull = fetch + merge
### CASE#4 - 가장 빈번한 경우
원격에 commit이력 발생. 동시에 로컬에 별도 커밋이 발생함.
문제는 같은 파일을 커밋한 경우이므로 push할 때 충돌 발생함.
해결방법
git pull origin main 하면 각각 커밋아이디가 있는 경우이므로
(참고로 case#5번은 한쪽만 커밋이 일어난 경우)
vscode GUI로 이미지 화면을 제공해줌.
틀린 부분에 대해서 병합파일 생성
병합파일 적절하게 수정했다면
git add .
git commit -m "메시지"
git push origin main

방법2
git fetch -> git diff -> git merge origin/main
충돌 해결한 후에 다시 
git push origin main


### CASE#5 
원격에 커밋 이력의 추가 발생 && working directory 또는 staging에서 같은 작업시
git pull origin main -> 충돌 발생
로컬에는 커밋 아이디가 없고 원격에는 커밋아이디가 있어서 
case4번과는 다르게 GUI로 화면제공이 안됨.

error: Your local changes to the following files: would be overwritten by merge

즉 
로컬에 커밋이력이 존재하지 않아서 merge할 수 있는 병합파일이 생성되지 않는 경우

방법1
로컬에서 커밋을 만든 후 git merge origin/main 충돌 제거 후
git add .
git commit -m "메시지"
git push orgin main

방법2
기존 수정본을 backup을 받아두고 로컬 작업 취소한다.
-> git pull origin main
-> backup파일을 참고하여 작업사항 번영

방법3
